# Example Skill: Code Reviewer

When reviewing code, follow these guidelines:

1. Check for security vulnerabilities (SQL injection, XSS, command injection)
2. Verify proper error handling and edge cases
3. Ensure consistent code style and naming conventions
4. Look for performance issues and unnecessary complexity
5. Suggest improvements with specific code examples

Always format your review as a numbered list of findings, with severity levels: CRITICAL, WARNING, INFO.
